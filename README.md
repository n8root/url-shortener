# Пет-проект 1: Сервис сокращения ссылок

## Описание проекта

Сервис для сокращения длинных URL-адресов с аналитикой кликов. Позволяет создавать короткие ссылки, отслеживать переходы и собирать статистику по каждой ссылке.

Бизнес-ценность:

- Внутренний инструмент для маркетинговых кампаний — отслеживание эффективности каналов
- Сокращение ссылок в email-рассылках, SMS, мессенджерах
- Аналитика переходов: география, источники, устройства
- Замена платных сервисов (Bitly, Short.io) собственным решением
- Контроль над данными — все метрики хранятся внутри компании

## Функциональные требования

### REST API эндпоинты

- POST /api/v1/shorten — создание короткой ссылки
  - Принимает JSON с полями: url (обязательно), custom_alias (опционально), expires_in (опционально, в секундах)
  - Возвращает JSON с полями: short_url, code, original_url, created_at, expires_at
  - Если custom_alias занят — возвращает ошибку 409 Conflict
  - Если URL невалиден — возвращает ошибку 400 Bad Request

- GET /:code — редирект по короткому коду
  - Возвращает HTTP 301 Moved Permanently с заголовком Location
  - Если код не найден — 404 Not Found
  - Если ссылка истекла — 410 Gone
  - Фиксирует клик асинхронно (не блокирует редирект)

- GET /api/v1/stats/:code — получение статистики по ссылке
  - Возвращает JSON: original_url, code, total_clicks, clicks_by_day (список), created_at, expires_at
  - Если код не найден — 404 Not Found

- DELETE /api/v1/urls/:code — удаление ссылки
  - Возвращает 204 No Content при успехе
  - Если код не найден — 404 Not Found

- GET /api/v1/health — проверка здоровья сервиса
  - Проверяет соединение с PostgreSQL и Redis
  - Возвращает JSON: status ("ok"/"degraded"/"down"), checks (список проверок с результатами)

### Валидация входных данных

- URL должен быть валидным (наличие схемы http/https, валидный хост)
- Максимальная длина URL — 2048 символов
- custom_alias — только латинские буквы, цифры, дефис, длина 3-32 символа
- expires_in — от 60 секунд до 365 дней

### Генерация коротких кодов

- Алгоритм: base62 (a-z, A-Z, 0-9)
- Длина кода: 6-8 символов
- Генерация через crypto/rand для криптографической случайности
- Проверка уникальности в базе перед сохранением
- При коллизии — повторная генерация (до 3 попыток)

### Поддержка пользовательских алиасов

- Пользователь может указать свой короткий код (custom_alias)
- Алиас проверяется на уникальность
- Зарезервированные слова (api, admin, health, stats) запрещены к использованию

### TTL (время жизни ссылки)

- По умолчанию ссылка без срока действия
- Можно указать expires_in в секундах при создании
- Просроченные ссылки возвращают 410 Gone при переходе
- Фоновая задача удаляет просроченные ссылки из кэша Redis

### Отслеживание кликов

- При каждом редиректе записываются:
  - IP-адрес клиента (с учётом X-Forwarded-For)
  - User-Agent
  - Referer
  - Временная метка клика
- Запись клика не должна замедлять редирект (асинхронная запись через горутину с буферизованным каналом)

### Rate limiting

- Ограничение по IP-адресу
- Лимит на создание ссылок: 10 запросов в минуту
- Лимит на редиректы: 100 запросов в минуту
- Реализация через Redis (sliding window counter)
- При превышении — 429 Too Many Requests с заголовком Retry-After

## Нефункциональные требования (метрики)

- Время отклика редиректа (GET /:code): < 10 мс (p99)
- Время отклика создания ссылки (POST /api/v1/shorten): < 50 мс (p99)
- Пропускная способность редиректов: 1000 RPS при p99 < 10 мс
- Доступность сервиса: 99.9% (не более 8.7 часов простоя в год)
- Вероятность коллизии короткого кода: < 0.001% при 10 миллионах ссылок
- Потеря кликов при асинхронной записи: < 0.01%
- Время старта сервиса: < 3 секунды
- Потребление памяти в idle: < 50 МБ

## Архитектура

### Слои Clean Architecture

- transport (handlers) — HTTP-обработчики, маршрутизация, валидация запросов, формирование ответов
- service (usecase) — бизнес-логика: создание ссылок, генерация кодов, обработка кликов
- repository — доступ к данным: PostgreSQL для хранения, Redis для кэша
- domain (entity) — структуры данных: URL, Click, ошибки домена

### PostgreSQL

- Таблица urls — хранение ссылок с индексами
- Таблица clicks — хранение кликов с индексом по url_id и clicked_at
- Миграции через goose

### Redis

- Кэш маппинга code -> original_url для быстрых редиректов
- TTL в Redis соответствует TTL ссылки
- При удалении ссылки — инвалидация кэша
- Sliding window counter для rate limiting

### Graceful shutdown

- Перехват сигналов SIGINT и SIGTERM
- Завершение текущих HTTP-запросов (http.Server.Shutdown)
- Ожидание записи буферизованных кликов
- Закрытие соединений с PostgreSQL и Redis
- Таймаут на graceful shutdown: 10 секунд

### Конфигурация

- Через переменные окружения
- Структура конфигурации: server (port, read_timeout, write_timeout), database (dsn, max_conns, min_conns), redis (addr, password, db), rate_limit (create_rpm, redirect_rpm)

## Схема базы данных

### Таблица urls

- id — BIGSERIAL PRIMARY KEY
- code — VARCHAR(32) NOT NULL UNIQUE
- original_url — TEXT NOT NULL
- custom_alias — BOOLEAN NOT NULL DEFAULT false
- created_at — TIMESTAMPTZ NOT NULL DEFAULT now()
- expires_at — TIMESTAMPTZ (nullable, NULL означает бессрочную ссылку)
- is_active — BOOLEAN NOT NULL DEFAULT true

Индексы:

- urls_code_idx — уникальный индекс по code (для быстрого поиска при редиректе)
- urls_expires_at_idx — индекс по expires_at WHERE expires_at IS NOT NULL (для фоновой очистки)
- urls_created_at_idx — индекс по created_at (для сортировки)

### Таблица clicks

- id — BIGSERIAL PRIMARY KEY
- url_id — BIGINT NOT NULL REFERENCES urls(id) ON DELETE CASCADE
- ip — INET NOT NULL
- user_agent — TEXT NOT NULL DEFAULT ''
- referer — TEXT NOT NULL DEFAULT ''
- clicked_at — TIMESTAMPTZ NOT NULL DEFAULT now()

Индексы:

- clicks_url_id_clicked_at_idx — составной индекс по (url_id, clicked_at) для быстрой выборки статистики

## API спецификация

### POST /api/v1/shorten

Тело запроса (JSON):

- url — строка, обязательное, оригинальный URL
- custom_alias — строка, опциональное, пользовательский короткий код
- expires_in — число, опциональное, время жизни в секундах

Тело ответа (JSON), статус 201 Created:

- short_url — строка, полный короткий URL (например "http://localhost:8080/abc123")
- code — строка, короткий код
- original_url — строка, оригинальный URL
- created_at — строка, время создания в формате RFC3339
- expires_at — строка или null, время истечения в формате RFC3339

Ошибки:

- 400 Bad Request — невалидный URL, слишком длинный URL, невалидный alias
- 409 Conflict — custom_alias уже занят
- 429 Too Many Requests — превышен лимит запросов

### GET /:code

Заголовки ответа, статус 301 Moved Permanently:

- Location — оригинальный URL

Ошибки:

- 404 Not Found — код не найден
- 410 Gone — ссылка истекла
- 429 Too Many Requests — превышен лимит запросов

### GET /api/v1/stats/:code

Тело ответа (JSON), статус 200 OK:

- code — строка, короткий код
- original_url — строка, оригинальный URL
- total_clicks — число, общее количество кликов
- clicks_by_day — список объектов: date (строка, "2024-01-15"), count (число)
- created_at — строка, время создания в формате RFC3339
- expires_at — строка или null, время истечения

Ошибки:

- 404 Not Found — код не найден

### DELETE /api/v1/urls/:code

Ответ, статус 204 No Content — тело отсутствует

Ошибки:

- 404 Not Found — код не найден

### GET /api/v1/health

Тело ответа (JSON), статус 200 OK:

- status — строка: "ok", "degraded" или "down"
- checks — список объектов:
  - name — строка ("postgresql", "redis")
  - status — строка ("ok", "down")
  - response_time_ms — число

### Формат ошибок (единый для всех эндпоинтов)

Тело ответа (JSON):

- error — строка, код ошибки (например "url_not_found", "invalid_url", "alias_taken")
- message — строка, человекочитаемое описание ошибки

## Требования к тестам

### Unit-тесты

- Валидация URL
  - Валидный HTTP URL проходит проверку
  - Валидный HTTPS URL проходит проверку
  - URL без схемы отклоняется
  - URL со схемой ftp отклоняется
  - Пустой URL отклоняется
  - URL длиннее 2048 символов отклоняется
  - URL с невалидным хостом отклоняется

- Генерация коротких кодов
  - Генерируется код длиной 6-8 символов
  - Код содержит только символы base62
  - 1000 последовательных генераций дают уникальные коды
  - Генерация использует crypto/rand (не math/rand)

- Валидация custom_alias
  - Валидный alias из латинских букв и цифр проходит проверку
  - Alias с дефисом проходит проверку
  - Alias короче 3 символов отклоняется
  - Alias длиннее 32 символов отклоняется
  - Alias с кириллицей отклоняется
  - Alias с пробелами отклоняется
  - Зарезервированные слова (api, admin, health) отклоняются

- Service layer (с замоканным репозиторием)
  - Создание ссылки: успешный сценарий
  - Создание ссылки: дубликат URL возвращает существующий код
  - Создание ссылки: коллизия кода — повторная генерация
  - Создание ссылки: custom_alias занят — ошибка 409
  - Получение ссылки по коду: успешный сценарий
  - Получение ссылки по коду: не найден — ошибка
  - Получение ссылки по коду: ссылка истекла — ошибка
  - Удаление ссылки: успешный сценарий
  - Удаление ссылки: не найден — ошибка
  - Получение статистики: корректный подсчёт кликов по дням

- Rate limiter
  - Первый запрос проходит
  - Запрос в пределах лимита проходит
  - Запрос сверх лимита блокируется
  - Лимит сбрасывается после окна времени
  - Разные IP имеют независимые лимиты

- TTL / истечение срока
  - Ссылка без TTL не истекает
  - Ссылка с TTL в будущем — активна
  - Ссылка с TTL в прошлом — истекла
  - Корректное вычисление expires_at из expires_in

### Интеграционные тесты

- Полный флоу: создание ссылки -> редирект -> проверка статистики
  - Создать короткую ссылку через POST /api/v1/shorten
  - Выполнить GET /:code, убедиться в 301 редиректе с правильным Location
  - Подождать запись клика (до 100 мс)
  - Проверить GET /api/v1/stats/:code — total_clicks == 1

- PostgreSQL операции (testcontainers)
  - Создание ссылки в базе и чтение по коду
  - Уникальность кода на уровне базы (UNIQUE constraint)
  - Каскадное удаление кликов при удалении ссылки
  - Конкурентная вставка с одним кодом — ровно одна запись успешна

- Redis кэш (testcontainers)
  - После создания ссылки — данные появляются в Redis
  - При редиректе — данные берутся из Redis (нет запроса к PostgreSQL)
  - При удалении ссылки — кэш инвалидируется
  - При истечении TTL в Redis — следующий запрос идёт в PostgreSQL

- Rate limiting с Redis
  - Отправить 10 запросов на POST /api/v1/shorten с одного IP — все проходят
  - Отправить 11-й запрос — получить 429 Too Many Requests
  - Проверить наличие заголовка Retry-After

- Обработка истекших ссылок
  - Создать ссылку с expires_in: 1
  - Подождать 2 секунды
  - GET /:code возвращает 410 Gone

- Конфликт custom_alias
  - Создать ссылку с custom_alias "mylink"
  - Попытаться создать вторую ссылку с тем же alias
  - Получить 409 Conflict с описанием ошибки

### Нагрузочные тесты

- Скрипт k6 для эндпоинта редиректа
  - Предварительно: создать 1000 коротких ссылок через API
  - Сценарий: случайный выбор кода из пула, GET /:code
  - Нагрузка: ramp-up от 0 до 1000 RPS за 30 секунд, удержание 1000 RPS 2 минуты
  - Критерии успеха:
    - p99 response time < 10 мс
    - Error rate < 0.1%
    - Все ответы — 301

- Скрипт k6 для эндпоинта создания ссылок
  - Сценарий: генерация уникальных URL, POST /api/v1/shorten
  - Нагрузка: 100 RPS, удержание 1 минуту
  - Критерии успеха:
    - p99 response time < 50 мс
    - Error rate < 0.1%
    - Все ответы — 201

## Чекпоинты реализации

1. Инициализация проекта — структура каталогов (cmd/, internal/, migrations/, configs/, scripts/), go.mod, Makefile с основными командами (build, run, test, lint, migrate), Docker Compose с PostgreSQL и Redis
2. Схема базы данных — миграции через goose: создание таблиц urls и clicks, индексы, файлы миграций в migrations/
3. Слой domain — структуры URL, Click, доменные ошибки (ErrNotFound, ErrAlreadyExists, ErrExpired), интерфейс репозитория
4. Слой repository — реализация PostgreSQL-репозитория (pgx), CRUD для urls, запись кликов, получение статистики по дням
5. Слой service — бизнес-логика: генерация коротких кодов, валидация URL и alias, создание/получение/удаление ссылок, обработка TTL
6. Слой transport (handlers) — chi-роутер, HTTP-обработчики для всех эндпоинтов, middleware для логирования и request_id
7. Redis кэширование — кэш URL-маппингов для редиректов, инвалидация при удалении, TTL кэша синхронизирован с TTL ссылки
8. Отслеживание кликов — асинхронная запись через буферизованный канал, горутина-воркер для батч-вставки кликов
9. Rate limiting — middleware на базе Redis sliding window counter, настраиваемые лимиты для разных эндпоинтов
10. Unit-тесты — тесты для валидации, генерации кодов, сервисного слоя (с моками через gomock или testify/mock)
11. Интеграционные тесты — testcontainers для PostgreSQL и Redis, тесты полного флоу, тесты конкурентности
12. Dockerfile — multi-stage build (builder -> scratch/distroless), минимальный размер образа
13. Health check эндпоинт — проверка PostgreSQL и Redis, формат ответа с деталями проверок
14. Prometheus метрики — http_requests_total, http_request_duration_seconds, urls_created_total, redirects_total, cache_hits_total, cache_misses_total
15. Нагрузочное тестирование — скрипты k6 для редиректов и создания ссылок, bash-скрипт для подготовки тестовых данных
